# Glide-HIMS Standalone Offline Deployment Guide

**For Government, Military, Remote Clinics, and Disaster Relief Operations**

---

## Overview

Glide-HIMS Standalone is a complete healthcare management system packaged for air-gapped (offline) deployment. It requires no internet connection and operates completely independently on customer infrastructure.

### Key Features

✅ **Zero Internet Required** - Fully offline operation
✅ **Single Container** - All-in-one Docker deployment
✅ **Embedded SQLite** - No external database server needed
✅ **Local License Validation** - No phone-home checks
✅ **Automatic Backups** - Daily backup to local storage
✅ **Manual Updates** - Control when you update
✅ **HIPAA/GDPR Ready** - Encryption, audit logs, data residency
✅ **Disaster Recovery** - One-command restore procedures

---

## System Requirements

### Minimum Requirements (for 1-100 patients/month)

- **CPU**: 2 cores (2.0 GHz+)
- **RAM**: 4 GB
- **Storage**: 20 GB total
  - 10 GB for application
  - 10 GB for patient data
- **Network**: None required (completely offline)
- **OS**: Linux (Ubuntu 20.04+, CentOS 8+, Red Hat 8+)

### Recommended (for 100-1000 patients/month)

- **CPU**: 4 cores (2.4 GHz+)
- **RAM**: 8-16 GB
- **Storage**: 100 GB total
  - 10 GB for application
  - 80 GB for patient data + backups
- **OS**: Linux (Ubuntu 22.04 LTS, CentOS Stream, Red Hat 9)

### Supported Platforms

- **Servers**: Any Linux server with Docker support
- **Laptops/Desktops**: Ubuntu, CentOS, Red Hat
- **Cloud VMs**: AWS EC2, Azure VM, Google Compute Engine (air-gapped setup)
- **Portable**: Can run on USB-connected external drives

## Quick Start (15 Minutes)

### Step 1: Prepare System

```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Install Docker (Ubuntu/Debian)
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Allow current user to run Docker (without sudo)
sudo usermod -aG docker $USER
newgrp docker

# Verify Docker
docker --version
```

### Step 2: Download Package

Option A: From USB/External Drive
```bash
# Copy from USB
cp /media/usb/glide-hims-standalone-v1.0.tar.gz ~/
cd ~
tar -xzf glide-hims-standalone-v1.0.tar.gz
cd glide-hims-standalone
```

Option B: From Download Server (if internet available once)
```bash
cd ~
wget https://downloads.glide-hims.io/standalone/v1.0/glide-hims-standalone-v1.0.tar.gz
tar -xzf glide-hims-standalone-v1.0.tar.gz
cd glide-hims-standalone
```

### Step 3: Run Installer

```bash
chmod +x install-standalone.sh
sudo ./install-standalone.sh
```

The installer will:
1. ✓ Check prerequisites (Docker installed, disk space)
2. ✓ Prompt for license key, admin password, hospital name
3. ✓ Create directories and configuration
4. ✓ Load Docker image
5. ✓ Start all containers
6. ✓ Verify installation

### Step 4: Access System

```bash
# Open browser to
http://localhost:3000

# Or from another computer
http://<server-ip>:3000
```

Log in with:
- **Username**: admin
- **Password**: (the one you set during installation)

## Configuration

### Environment Variables

Edit `/opt/glide-hims/.env`:

```bash
# Database
DB_TYPE=sqlite
SQLITE_PATH=/data/glide-hims.db

# Security
JWT_SECRET=<auto-generated>
JWT_EXPIRATION=24h
ENABLE_2FA=false

# Organization
HOSPITAL_NAME=Your Hospital Name
LICENSE_KEY=GLI-STD-XXXXXXXX

# Backups
BACKUP_LOCATION=/data/glide-hims-backups
BACKUP_SCHEDULE="0 2 * * *"  # Daily 2 AM

# Updates
UPDATES_PATH=/data/glide-hims-updates
CHECK_UPDATES_INTERVAL=604800  # Weekly
```

### Apply Configuration Changes

```bash
# Edit .env file
nano /opt/glide-hims/.env

# Restart containers
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml restart

# Check status
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml ps
```

## Daily Operations

### Check System Status

```bash
# View all containers
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml ps

# View logs (last 50 lines)
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml logs --tail=50

# Follow logs in real-time
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml logs -f
```

### Start/Stop System

```bash
# Start system
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml start

# Stop system
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml stop

# Restart all containers
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml restart

# Stop and remove containers (keep data)
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml down
```

### View System Resources

```bash
# CPU and memory usage
docker stats glide-hims-backend

# Database file size
du -h /data/glide-hims/glide-hims.db

# Total storage used
du -h /data/glide-hims*/
```

## Backup & Recovery

### Automatic Backups

Backups run automatically every day at 2 AM and are stored at:
```
/data/glide-hims-backups/
```

Each backup includes:
- SQLite database
- All patient data
- Configuration
- Audit logs

### Manual Backup

```bash
# Create backup right now
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml exec backend \
  sh -c 'sqlite3 /data/glide-hims.db ".backup /data/glide-hims-backups/manual-backup-$(date +%Y%m%d-%H%M%S).db"'

# List all backups
ls -lh /data/glide-hims-backups/

# Backup to USB drive
cp /data/glide-hims-backups/* /media/usb/glide-hims-backups/
```

### Restore from Backup

```bash
# List available backups
ls -lh /data/glide-hims-backups/

# Restore from backup (STOPS THE SYSTEM)
sudo systemctl stop glide-hims.service  # if using systemd
# Or:
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml stop

# Restore the database
cp /data/glide-hims-backups/backup-2026-04-26.db /data/glide-hims/glide-hims.db

# Start system again
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml start

# Verify data
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml logs backend
```

## Updates (Offline)

### Check for Updates

```bash
# Check for available updates
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml exec backend \
  npm run check-updates
```

### Apply Updates

```bash
# Copy update file from USB
cp /media/usb/glide-hims-v1.1.update /data/glide-hims-updates/

# Apply update (system stays running)
docker-compose -f /opt/glide-hims/docker-compose.standalone.yml exec backend \
  npm run apply-update -- glide-hims-v1.1.update
```
