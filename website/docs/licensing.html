Licensing System

Enterprise license management and validation

✓ Verified in v1.0 — license keys are issued from the System Administration portal when provisioning a hybrid or standalone deployment. Keys are cryptographically signed and encode tenant id, plan, user/facility limits, and expiry. Validation runs periodically; expired keys gracefully transition the deployment into a restricted state until a valid key is installed.
Overview

Glide HIMS uses a cryptographically signed licensing system to manage enterprise deployments. Each license key encodes:

  • Organization details - Company name and tenant ID
  • License type - Determines available features
  • User limits - Maximum concurrent users
  • Facility limits - Maximum hospitals/clinics
  • Enabled modules - Which system modules are active
  • Expiration date - License validity period
  • Hardware binding - Optional server fingerprint lock

Activating Your License
Step 1: Set the License Key

Add your license key to the environment configuration:

# In your .env file
LICENSE_KEY=GLIDE-XXXX-XXXX-XXXX-XXXX

# Or set as environment variable
export LICENSE_KEY="GLIDE-XXXX-XXXX-XXXX-XXXX"
Step 2: Validate the License

The system validates the license on startup, but you can also validate manually:

# Validate license via API
curl -X POST https://your-server/api/v1/license/validate \
  -H "Content-Type: application/json" \
  -d '{"licenseKey": "GLIDE-XXXX-XXXX-XXXX-XXXX"}'

# Response (valid license)
{
  "valid": true,
  "expiresAt": "2027-04-10T00:00:00.000Z",
  "expiresIn": 365,
  "licenseType": "enterprise",
  "maxUsers": 500,
  "maxFacilities": 10,
  "enabledModules": ["patients", "billing", "pharmacy", "laboratory", "ipd", "hr"]
}
Step 3: Hardware Binding (Optional)

For on-premise deployments, you can bind the license to specific hardware:

# Get your server's hardware ID
curl https://your-server/api/v1/phone-home/system-info \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

# Response
{
  "hardwareId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "systemInfo": {
    "hostname": "hims-server-01",
    "platform": "linux",
    "cpus": 8,
    "totalMemory": 17179869184
  }
}

# Bind license to hardware
curl -X PUT https://your-server/api/v1/license/GLIDE-XXXX/bind-hardware \
  -H "Content-Type: application/json" \
  -d '{"hardwareId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"}'

How It Works (Code Reference)

The licensing system uses HMAC-SHA256 signatures to ensure license integrity:

// License Service - Key Generation
import * as crypto from 'crypto';

export class LicenseService {
  private readonly secretKey: string;

  constructor() {
    this.secretKey = process.env.LICENSE_SECRET_KEY || 'default-secret';
  }

  /**
   * Generate a cryptographically signed license key
   */
  async generateLicense(dto: GenerateLicenseDto): Promise {
    // Create license data
    const licenseData = {
      org: dto.organizationName,
      type: dto.licenseType,
      users: dto.maxUsers,
      facilities: dto.maxFacilities,
      modules: dto.enabledModules,
      exp: Date.now() + (dto.validityDays * 24 * 60 * 60 * 1000),
    };

    // Generate signature
    const dataString = JSON.stringify(licenseData);
    const signature = crypto
      .createHmac('sha256', this.secretKey)
      .update(dataString)
      .digest('hex');

    // Create license key format: PREFIX-DATA-SIGNATURE
    const encoded = Buffer.from(dataString).toString('base64url');
    const licenseKey = `GLIDE-${encoded}-${signature.substring(0, 16)}`;

    // Store in database
    const license = this.licenseRepository.create({
      licenseKey,
      organizationName: dto.organizationName,
      licenseType: dto.licenseType,
      maxUsers: dto.maxUsers,
      signature,
      // ... other fields
    });

    return this.licenseRepository.save(license);
  }

  /**
   * Validate a license key
   */
  async validateLicense(licenseKey: string): Promise {
    // Check cache first (24-hour TTL)
    const cached = this.validationCache.get(licenseKey);
    if (cached) return cached;

    // Find license in database
    const license = await this.licenseRepository.findOne({
      where: { licenseKey },
    });

    if (!license) {
      return { valid: false, error: 'License not found' };
    }

    // Check status
    if (license.status === 'revoked') {
      return { valid: false, error: 'License has been revoked' };
    }

    // Check expiration
    if (new Date() > license.expiresAt) {
      return { valid: false, error: 'License has expired' };
    }

    // Verify signature
    const isSignatureValid = this.verifySignature(license);
    if (!isSignatureValid) {
      return { valid: false, error: 'Invalid license signature' };
    }

    // Check hardware binding
    if (license.hardwareId) {
      const currentHardwareId = this.getHardwareId();
      if (license.hardwareId !== currentHardwareId) {
        return { valid: false, error: 'Hardware mismatch' };
      }
    }

    const result = {
      valid: true,
      license,
      expiresIn: Math.ceil(
        (license.expiresAt.getTime() - Date.now()) / (1000 * 60 * 60 * 24)
      ),
    };

    // Cache the result
    this.validationCache.set(licenseKey, result, 24 * 60 * 60 * 1000);

    return result;
  }
}
Troubleshooting License Issues
Common Errors
ErrorCauseSolution
License not found Invalid or mistyped license key Verify the license key is correct. Check for typos.
License has expired License validity period ended Contact IT Solutions Uganda to renew your license.
License has been revoked License was manually revoked Contact support to understand why and get a new license.
Hardware mismatch Server hardware changed Contact support to rebind the license to new hardware.
Invalid license signature License data was tampered with Obtain a fresh license key from your account portal.
User limit exceeded More users than license allows Upgrade your license or deactivate unused user accounts.
