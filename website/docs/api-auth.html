Authentication API

JWT-based authentication with multi-tenant support

✓ Verified in v1.0 — tokens are issued as HttpOnly cookies (not in JSON), so XSS cannot exfiltrate them. accessToken is scoped to / (15 min); refreshToken is scoped to /api/v1/auth/refresh (7 days). The frontend writes a non-sensitive glide_login_kind=system|tenant flag in localStorage to route 401 redirects to the correct login page (/system/login vs /login/{tenant-slug}). System administrators authenticate via the same POST /api/v1/auth/login endpoint and are identified by user.isSystemAdmin === true.

Overview

Glide HIMS uses JWT (JSON Web Tokens) for authentication. The system supports:

  • Multi-tenant authentication - Users belong to specific tenants (clinics/hospitals)
  • Role-based access control (RBAC) - Permissions based on user roles
  • Token refresh - Seamless session extension without re-login
  • Secure password handling - bcrypt hashing with salt

Endpoints

POST /api/v1/auth/login

Authenticate user and receive JWT tokens.

Request Body
Parameter Type Description
email required string User's email address
password required string User's password
tenantId required string Tenant identifier (e.g., "amani-childrens-clinic")
rememberMe optional boolean Extend refresh token expiry to 30 days
Example Request
curl -X POST https://your-domain.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "doctor@clinic.com",
    "password": "securePassword123",
    "tenantId": "amani-childrens-clinic"
  }'
Success Response (200 OK)
{
  "success": true,
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 900,
    "tokenType": "Bearer",
    "user": {
      "id": "usr_abc123",
      "email": "doctor@clinic.com",
      "firstName": "John",
      "lastName": "Doe",
      "role": "doctor",
      "permissions": ["patients:read", "patients:write", "appointments:read"],
      "tenantId": "amani-childrens-clinic",
      "tenantName": "Amani Children's Clinic"
    }
  }
}

Using Authentication

Making Authenticated Requests

Include the access token in the Authorization header:

# All API requests require the Bearer token
curl -X GET https://your-domain.com/api/v1/patients \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
Frontend Integration (React) 
// lib/auth.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import api from './api';

interface AuthState {
  user: User | null;
  accessToken: string | null;
  isAuthenticated: boolean;
  login: (email: string, password: string, tenantId: string) => Promise<void>;
  logout: () => Promise<void>;
  refreshToken: () => Promise<void>;
}

export const useAuth = create<AuthState>()(
  persist(
    (set, get) => ({
      user: null,
      accessToken: null,
      isAuthenticated: false,

      login: async (email, password, tenantId) => {
        const response = await api.post('/auth/login', {
          email, password, tenantId
        });
        
        const { accessToken, refreshToken, user } = response.data.data;
        
        // Store access token in memory/state
        set({ user, accessToken, isAuthenticated: true });
        
        // Store refresh token securely (httpOnly cookie preferred)
        localStorage.setItem('refreshToken', refreshToken);
        
        // Set up axios interceptor for authenticated requests
        api.defaults.headers.common['Authorization'] = `Bearer ${accessToken}`;
      },

      logout: async () => {
        try {
          await api.post('/auth/logout');
        } finally {
          set({ user: null, accessToken: null, isAuthenticated: false });
          localStorage.removeItem('refreshToken');
          delete api.defaults.headers.common['Authorization'];
        }
      },

      refreshToken: async () => {
        const refreshToken = localStorage.getItem('refreshToken');
        if (!refreshToken) throw new Error('No refresh token');
        
        const response = await api.post('/auth/refresh', { refreshToken });
        const { accessToken } = response.data.data;
        
        set({ accessToken });
        api.defaults.headers.common['Authorization'] = `Bearer ${accessToken}`;
      }
    }),
    { name: 'auth-storage', partialize: (state) => ({ user: state.user }) }
  )
);
Auto Token Refresh (Axios Interceptor) 
// lib/api.ts
import axios from 'axios';
import { useAuth } from './auth';

const api = axios.create({
  baseURL: '/api/v1',
  timeout: 10000,
});

// Response interceptor for token refresh
api.interceptors.response.use(
  (response) => response,
  async (error) => {
    const originalRequest = error.config;
    
    // If 401 and not already retrying
    if (error.response?.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true;
      
      try {
        await useAuth.getState().refreshToken();
        
        // Retry original request with new token
        const token = useAuth.getState().accessToken;
        originalRequest.headers['Authorization'] = `Bearer ${token}`;
        return api(originalRequest);
      } catch (refreshError) {
        // Refresh failed, logout user
        useAuth.getState().logout();
        window.location.href = '/login';
        return Promise.reject(refreshError);
      }
    }
    
    return Promise.reject(error);
  }
);

export default api;

Security Best Practices

Recommended
  • Store refresh tokens in HttpOnly cookies
  • Use short-lived access tokens (15 min)
  • Implement rate limiting on auth endpoints
  • Use HTTPS in production
  • Rotate JWT secrets periodically
  • Log all authentication events
Avoid
  • Storing tokens in localStorage (XSS vulnerable)
  • Long-lived access tokens
  • Exposing JWT secret in client code
  • Ignoring token expiry
  • Using weak passwords
  • Skipping CSRF protection
