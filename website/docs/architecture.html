Request Flow
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Client  │───▶│   Nginx  │───▶│  Guards  │───▶│Controller│───▶│ Service  │
│ (Browser)│    │  (SSL)   │    │  (Auth)  │    │ (Route)  │    │ (Logic)  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └────┬─────┘
                                                                     │
                                                                     ▼
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ Response │◀───│Interceptor◀───│ Service  │◀───│Repository│◀───│ Database │
│  (JSON)  │    │(Transform)│    │ (Logic)  │    │ (ORM)    │    │  (PG)    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘

Flow explanation: 1. Client sends request → 2. Nginx handles SSL termination and proxies to backend → 3. Auth guards validate JWT token → 4. Controller handles route → 5. Service contains business logic → 6. Repository interacts with database → 7. Response transformed and returned to client

Security Architecture
Authentication
  • JWT-based authentication
  • Access tokens (1 hour expiry)
  • Refresh tokens (7 days expiry)
  • Optional MFA (TOTP)
  • Biometric authentication support
Authorization
  • Role-based access control (RBAC)
  • Permission-based guards
  • Tenant isolation (multi-tenant)
  • Facility-level access control
  • Resource ownership checks
Data Protection
  • TLS 1.3 for transport encryption
  • Passwords hashed with bcrypt
  • Sensitive data encryption at rest
  • Audit logging for all changes
API Security
  • Rate limiting (100 req/min default)
  • CORS configuration
  • Input validation (class-validator)
  • SQL injection prevention (TypeORM)
  • XSS protection headers
