Monitoring & Observability

Track system health, performance, and usage metrics

✓ Verified in v1.0 (Phase 4) — the platform ships with: a deep /health endpoint that probes PostgreSQL (SELECT 1) and the Redis cache (set/get round-trip); HealthMetricsCollectorService for real-time metric collection with anomaly detection; and an AlertingService with email, SMS, Slack, PagerDuty, and webhook channels. See live /health on the demo.

Overview

Effective monitoring ensures your Glide HIMS deployment runs smoothly. This guide covers:

  • Health checks - Verify system components are operational
  • Metrics - Track performance and resource usage
  • Logging - Capture and analyze application logs
  • Alerting - Get notified of issues before they impact users

Health Check Endpoints

GET /api/v1/health

Basic health check - returns system status and component health.

# Request
curl https://your-domain.com/api/v1/health

# Response (200 OK)
{
  "status": "healthy",
  "timestamp": "2026-04-10T00:54:00.000Z",
  "version": "1.0.0",
  "uptime": 86400,
  "components": {
    "database": {
      "status": "healthy",
      "latency": 5
    },
    "redis": {
      "status": "healthy",
      "latency": 2
    },
    "storage": {
      "status": "healthy",
      "freeSpace": "45GB"
    }
  }
}
GET /api/v1/health/live

Liveness probe - simple check if the application is running.

# Response (200 OK)
{ "status": "ok" }

# Response (503 Service Unavailable)
{ "status": "error", "message": "Application not ready" }
GET /api/v1/health/ready

Readiness probe - checks if the application can handle requests.

# Response (200 OK)
{
  "status": "ready",
  "checks": {
    "database": true,
    "redis": true,
    "migrations": true
  }
}

Alerting Rules

High Error Rate

Trigger when error rate exceeds 5% for 5 minutes

- alert: HighErrorRate
  expr: rate(http_request_errors_total[5m]) / rate(http_requests_total[5m]) > 0.05
  for: 5m
  labels:
    severity: critical
  annotations:
    summary: "High error rate detected"
    description: "Error rate is {{ $value | humanizePercentage }}"
Database Connection Failure

Trigger when database is unreachable

- alert: DatabaseDown
  expr: pg_up == 0
  for: 1m
  labels:
    severity: critical
  annotations:
    summary: "PostgreSQL is down"
    description: "Database connection failed for more than 1 minute"
High Response Time

Trigger when p95 response time exceeds 2 seconds

- alert: HighResponseTime
  expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 2
  for: 10m
  labels:
    severity: warning
  annotations:
    summary: "High API response time"
    description: "95th percentile response time is {{ $value }}s"
Disk Space Low

Trigger when disk usage exceeds 80%

- alert: DiskSpaceLow
  expr: (node_filesystem_size_bytes - node_filesystem_free_bytes) / node_filesystem_size_bytes > 0.8
  for: 15m
  labels:
    severity: warning
  annotations:
    summary: "Disk space running low"
    description: "Disk usage is {{ $value | humanizePercentage }}"

Quick Health Check Script

#!/bin/bash
# health-monitor.sh - Simple monitoring script

DOMAIN="https://your-domain.com"
ALERT_EMAIL="admin@your-domain.com"
LOG_FILE="/var/log/glide-hims/health-check.log"

check_health() {
    local response=$(curl -s -o /dev/null -w "%{http_code}" "$DOMAIN/api/v1/health")
    local timestamp=$(date '+%Y-%m-%d %H:%M:%S')
    
    if [ "$response" != "200" ]; then
        echo "[$timestamp] CRITICAL: Health check failed (HTTP $response)" >> $LOG_FILE
        echo "Glide HIMS health check failed at $timestamp (HTTP $response)" | \
            mail -s "ALERT: Glide HIMS Down" $ALERT_EMAIL
        return 1
    else
        echo "[$timestamp] OK: Health check passed" >> $LOG_FILE
        return 0
    fi
}

check_disk() {
    local usage=$(df -h / | awk 'NR==2 {print $5}' | tr -d '%')
    if [ "$usage" -gt 80 ]; then
        echo "WARNING: Disk usage at ${usage}%" >> $LOG_FILE
    fi
}

check_memory() {
    local usage=$(free | awk '/Mem:/ {printf "%.0f", $3/$2 * 100}')
    if [ "$usage" -gt 85 ]; then
        echo "WARNING: Memory usage at ${usage}%" >> $LOG_FILE
    fi
}

# Run checks
check_health
check_disk
check_memory

# Add to crontab for every 5 minutes:
# */5 * * * * /path/to/health-monitor.sh
