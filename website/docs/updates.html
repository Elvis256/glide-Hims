Update System

Managing system updates and version control

Overview

Glide HIMS includes a built-in update management system that:

  • Checks for available updates from the central server
  • Downloads and verifies update packages
  • Provides release notes and changelogs
  • Supports both automatic and manual update workflows
  • Tracks mandatory security updates
Checking for Updates
Via API
# Check if updates are available
curl https://hmisdemo.itsolutionsuganda.com/api/v1/updates/check?version=1.0.0

# Response when update available
{
  "statusCode": 200,
  "data": {
    "updateAvailable": true,
    "currentVersion": "1.0.0",
    "latestVersion": "1.1.0",
    "isMandatory": false,
    "releaseNotes": "Bug fixes and performance improvements"
  }
}

# Response when up to date
{
  "statusCode": 200,
  "data": {
    "updateAvailable": false,
    "currentVersion": "1.1.0"
  }
}
Via Command Line
# Using the built-in update script
cd /opt/glide-hims
./scripts/check-updates.sh

# Output:
# Current version: 1.0.0
# Latest version: 1.1.0
# Update available: Yes
# Mandatory: No

Manual Update Process
Step 1: Backup Current Installation
# Create backup
./backup.sh

# Or manually
pg_dump -U glide_admin glide_hims > backup_$(date +%Y%m%d).sql
tar -czf app_backup_$(date +%Y%m%d).tar.gz /opt/glide-hims
Step 2: Download Update
# Download from API
curl -o glide-hims-1.1.0.tar.gz \
  "https://hmisdemo.itsolutionsuganda.com/api/v1/updates/download/1.1.0"

# Verify checksum (if provided)
sha256sum glide-hims-1.1.0.tar.gz
# Compare with checksum from version info
Step 3: Stop Services
# Using PM2
pm2 stop glide-hims-backend glide-hims-frontend

# Or systemd
sudo systemctl stop glide-hims-backend
Step 4: Apply Update
# Extract update
tar -xzf glide-hims-1.1.0.tar.gz -C /opt/glide-hims

# Install new dependencies
cd /opt/glide-hims
pnpm install

# Run database migrations
cd packages/backend
pnpm run migration:run

# Build applications
cd /opt/glide-hims
pnpm run build
Step 5: Restart Services
# Using PM2
pm2 restart glide-hims-backend glide-hims-frontend

# Verify
pm2 status
curl https://yourserver/api/v1/health
Tip: Always test updates in a staging environment before applying to production.
Automated Update Script
#!/bin/bash
# update-glide-hims.sh - Automated update script

set -e

INSTALL_DIR="/opt/glide-hims"
BACKUP_DIR="/var/backups/glide-hims"
UPDATE_URL="https://hmisdemo.itsolutionsuganda.com/api/v1/updates"

# Colors for output
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

echo -e "${GREEN}=== Glide HIMS Update Script ===${NC}"

# Get current version
CURRENT_VERSION=$(cat $INSTALL_DIR/package.json | grep '"version"' | head -1 | awk -F'"' '{print $4}')
echo "Current version: $CURRENT_VERSION"

# Check for updates
echo "Checking for updates..."
UPDATE_INFO=$(curl -s "$UPDATE_URL/check?version=$CURRENT_VERSION")
UPDATE_AVAILABLE=$(echo $UPDATE_INFO | jq -r '.data.updateAvailable')

if [ "$UPDATE_AVAILABLE" != "true" ]; then
    echo -e "${GREEN}System is up to date!${NC}"
    exit 0
fi

LATEST_VERSION=$(echo $UPDATE_INFO | jq -r '.data.latestVersion')
IS_MANDATORY=$(echo $UPDATE_INFO | jq -r '.data.isMandatory')

echo -e "${YELLOW}Update available: $CURRENT_VERSION -> $LATEST_VERSION${NC}"
if [ "$IS_MANDATORY" == "true" ]; then
    echo -e "${RED}This is a MANDATORY security update!${NC}"
fi

# Confirm update
read -p "Do you want to proceed with the update? (y/n) " -n 1 -r
echo
if [[ ! $REPLY =~ ^[Yy]$ ]]; then
    echo "Update cancelled."
    exit 0
fi

# Create backup
echo "Creating backup..."
mkdir -p $BACKUP_DIR
BACKUP_FILE="$BACKUP_DIR/backup_$(date +%Y%m%d_%H%M%S).tar.gz"
pg_dump -U glide_admin glide_hims | gzip > "$BACKUP_DIR/db_$(date +%Y%m%d_%H%M%S).sql.gz"
tar -czf $BACKUP_FILE $INSTALL_DIR --exclude='node_modules'
echo "Backup saved to: $BACKUP_FILE"

# Stop services
echo "Stopping services..."
pm2 stop glide-hims-backend glide-hims-frontend

# Download update
echo "Downloading update..."
curl -o /tmp/glide-hims-$LATEST_VERSION.tar.gz "$UPDATE_URL/download/$LATEST_VERSION"

# Apply update
echo "Applying update..."
tar -xzf /tmp/glide-hims-$LATEST_VERSION.tar.gz -C $INSTALL_DIR

# Install dependencies
echo "Installing dependencies..."
cd $INSTALL_DIR && pnpm install

# Run migrations
echo "Running database migrations..."
cd $INSTALL_DIR/packages/backend && pnpm run migration:run

# Build
echo "Building applications..."
cd $INSTALL_DIR && pnpm run build

# Restart services
echo "Restarting services..."
pm2 restart glide-hims-backend glide-hims-frontend

# Verify
echo "Verifying update..."
sleep 5
HEALTH=$(curl -s http://localhost:3000/api/v1/health | jq -r '.status')
if [ "$HEALTH" == "ok" ]; then
    echo -e "${GREEN}Update successful! Now running version $LATEST_VERSION${NC}"
else
    echo -e "${RED}Health check failed. Rolling back...${NC}"
    # Rollback logic here
    exit 1
fi

# Cleanup
rm /tmp/glide-hims-$LATEST_VERSION.tar.gz

echo "Update complete!"
Version Information Structure
// Version object structure
{
  "id": "uuid",
  "version": "1.1.0",
  "versionCode": "20260415",
  "releaseNotes": "Full markdown release notes...",
  "minUpgradeFrom": "1.0.0",      // Minimum version that can upgrade to this
  "isMandatory": false,            // Required security update
  "isLatest": true,                // Currently marked as latest stable
  "downloadUrl": "https://...",    // External download URL (optional)
  "checksum": "sha256:abc123...",  // SHA-256 checksum for verification
  "fileSize": 52428800,            // Size in bytes
  "releasedAt": "2026-04-15T00:00:00.000Z",
  "endOfSupport": null             // When this version stops receiving updates
}
Version Numbering

Glide HIMS follows Semantic Versioning:

  • MAJOR (1.x.x) - Breaking changes, major new features
  • MINOR (x.1.x) - New features, backward compatible
  • PATCH (x.x.1) - Bug fixes, security patches
Rollback Procedure

If an update causes issues, you can rollback to the previous version:

# Stop services
pm2 stop all

# Restore database
gunzip -c /var/backups/glide-hims/db_YYYYMMDD.sql.gz | psql -U glide_admin glide_hims

# Restore application files
tar -xzf /var/backups/glide-hims/backup_YYYYMMDD.tar.gz -C /

# Rebuild (in case node_modules changed)
cd /opt/glide-hims
pnpm install
pnpm run build

# Restart services
pm2 restart all

# Verify
curl https://yourserver/api/v1/health
Warning: Database rollback will lose any data created after the backup. Consider the impact before rolling back.
