Troubleshooting Guide

Common issues and their solutions

Quick Diagnostics

Run these commands to quickly diagnose common issues:

# Check if backend is running
pm2 status glide-hims-backend

# Check backend logs
pm2 logs glide-hims-backend --lines 50

# Check database connection
psql -h localhost -U glide_admin -d glide_hims -c "SELECT 1;"

# Test API health
curl -k https://localhost:3000/api/v1/health

# Check disk space
df -h

# Check memory usage
free -m

# Check port usage
netstat -tlnp | grep -E "3000|5432|6379"

Common Issues

Symptoms:
  • PM2 shows backend status as "errored" or keeps restarting
  • Error logs show "Cannot connect to database" or "ECONNREFUSED"
Possible Causes & Solutions:
Database not running
# Check if PostgreSQL is running
systemctl status postgresql

# Start PostgreSQL
sudo systemctl start postgresql

# Or with Docker
docker-compose up -d postgres
Wrong database credentials
# Check your .env file
cat .env | grep DB_

# Test connection manually
psql -h $DB_HOST -U $DB_USER -d $DB_NAME

# If password is wrong, reset it
sudo -u postgres psql -c "ALTER USER glide_admin WITH PASSWORD 'new_password';"
Port already in use
# Find what's using port 3000
lsof -i :3000

# Kill the process (replace PID)
kill -9 <PID>

# Or change the port in .env
PORT=3001
Missing environment variables
# Check required variables are set
node -e "console.log(require('dotenv').config().parsed)"

# Minimum required variables:
# DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD
# JWT_SECRET

Symptoms:
  • API returns {"statusCode":401,"error":"Unauthorized"}
  • Users can't log in
  • Token refresh fails
Token expired

Access tokens expire after 1 hour by default. Use the refresh token:

curl -X POST https://api.example.com/api/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refreshToken": "your-refresh-token"}'
JWT_SECRET changed

If the JWT_SECRET was changed, all existing tokens are invalidated:

# Check JWT_SECRET is consistent across restarts
echo $JWT_SECRET

# Users need to log in again after secret change
Missing Authorization header
# Correct format
curl -H "Authorization: Bearer YOUR_TOKEN" https://api/endpoint

# Common mistakes:
# ❌ Authorization: YOUR_TOKEN (missing "Bearer")
# ❌ Authentication: Bearer YOUR_TOKEN (wrong header name)
# ❌ Bearer: YOUR_TOKEN (wrong format)

Connection pool exhausted

Too many connections. Check and adjust pool settings:

# Check current connections
psql -c "SELECT count(*) FROM pg_stat_activity WHERE datname='glide_hims';"

# Increase max connections in postgresql.conf
# max_connections = 200

# Or adjust TypeORM pool size in .env
DB_POOL_SIZE=20
Migration not run
# Run pending migrations
cd packages/backend
pnpm run migration:run

# Or run manually
psql -f src/database/migrations/*.sql
Disk full
# Check disk space
df -h

# Clean old logs
sudo journalctl --vacuum-time=7d

# Clean Docker
docker system prune -a

Symptoms:
  • White screen or error boundary message
  • Console shows JavaScript errors
  • TypeError: Cannot read properties of undefined
Clear browser cache

After updates, cached JavaScript may conflict:

# Hard refresh in browser
Ctrl+Shift+R (Windows/Linux)
Cmd+Shift+R (Mac)

# Or clear all site data in DevTools
Application > Storage > Clear site data
Rebuild frontend
cd packages/frontend
pnpm run build

# Restart the server
pm2 restart glide-hims-frontend
Check API connectivity

Frontend can't reach backend:

# Check VITE_API_URL in frontend .env
echo $VITE_API_URL

# Should match your backend URL
VITE_API_URL=https://your-domain.com/api/v1

License expired
# Check license status
curl -X POST https://localhost:3000/api/v1/license/validate \
  -H "Content-Type: application/json" \
  -d '{"licenseKey": "YOUR-KEY"}'

# Contact support to renew
Hardware changed

If the server hardware changed, the license binding is invalid:

# Get new hardware ID
curl https://localhost:3000/api/v1/phone-home/system-info \
  -H "Authorization: Bearer TOKEN"

# Contact support with the new hardware ID to rebind
Clock skew

Server time must be accurate for license validation:

# Check server time
date

# Sync with NTP
sudo timedatectl set-ntp true

# Or manually set timezone
sudo timedatectl set-timezone Africa/Kampala

Check system resources
# CPU and memory
htop

# Disk I/O
iotop

# Network
nethogs
Database optimization
# Check slow queries
psql -c "SELECT query, calls, mean_time FROM pg_stat_statements ORDER BY mean_time DESC LIMIT 10;"

# Run VACUUM
psql -c "VACUUM ANALYZE;"

# Check table sizes
psql -c "SELECT relname, pg_size_pretty(pg_total_relation_size(relid)) FROM pg_catalog.pg_statio_user_tables ORDER BY pg_total_relation_size(relid) DESC;"
Enable caching
# Ensure Redis is running
docker-compose up -d redis

# Check Redis connection in .env
REDIS_HOST=localhost
REDIS_PORT=6379
