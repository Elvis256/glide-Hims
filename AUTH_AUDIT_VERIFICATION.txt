REQUIREMENT 4: PERMISSION RESOLUTION
✅ How system determines user has permission - DOCUMENTED
   
   Load user roles:
   ├─ SELECT * FROM user_roles WHERE user_id = $1
   ├─ Filters by facility if provided
   └─ Documented: AUTH_AUDIT_COMPREHENSIVE.md Section 4.3 Step 1
   
   Get role permissions:
   ├─ SELECT * FROM role_permissions JOIN permissions
   ├─ WHERE role_id IN (user's role ids)
   └─ Documented: AUTH_AUDIT_COMPREHENSIVE.md Section 4.3 Step 2
   
   Get direct user permissions:
   ├─ SELECT * FROM user_permissions JOIN permissions
   ├─ WHERE user_id = $1
   └─ Documented: AUTH_AUDIT_COMPREHENSIVE.md Section 4.3 Step 3
   
   Combine and check:
   ├─ requiredPermissions.every(p => userHas.includes(p))
   └─ Documented: AUTH_AUDIT_COMPREHENSIVE.md Section 4.3 Step 4

✅ JWT vs Database lookup - DOCUMENTED
   Status: PERMISSIONS NOT IN JWT (loads from database)
   Why: Immediate permission changes without re-login
   Trade-off: Extra DB queries per request
   Documented: AUTH_AUDIT_COMPREHENSIVE.md Section 4.2
   JWT has: sub, username, email, roles (names only), tenantId, facilityId
   JWT doesn't have: Permission codes, permission IDs
